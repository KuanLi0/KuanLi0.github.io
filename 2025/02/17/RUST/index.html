RUST的变量

  1. rust中的变量选择了手动设定变量的可变性

  2. 变量的命名和其他语言一样，遵循规则就行

  3. 变量绑定：类似于变量的赋值，这里涉及到了rust的核心原则–所有权（任何内存对象都是有主人的），绑定就是把这个对象绑定给一个变量，原来的主任就会丧失该内存对象的所有权。

  4. 变量的可变性：在默认的情况下变量一般是不可变的，可以使用mut关键字进行标识，让变量可变。改变时并不会创建一个新的内存对象。

  5. 在声明一个变量时，在变量的名字开头可以加上 _ ，这样编译器就不会发出警告。

  6. 常量：用const进行修饰，而不用let。从始至终不允许改变，且声明时必须进行类型的说明。

  7. 变量遮蔽：RUST中允许声明相同名字的变量，但后面的变量会遮蔽掉前面的变量。

    RUST变量类型

    rust变量类型分为基本类型和符合类型

    ​ 基本类型包括

    • 数值类型: 有符号整数 (i8, i16, i32, i64, isize)、 无符号整数 (u8, u16, u32, u64, usize) 、浮点数 (f32, f64)、以及有理数、复数
    • 字符串：字符串字面量和字符串切片 &str
    • 布尔类型： true和false
    • 字符类型: 表示单个 Unicode 字符，存储为 4 个字节
    • 单元类型: 即 () ，其唯一的值也是 ()

rust中的string类型分配在堆上。它由栈上的堆指针，字符串长度和字符串容量构成。

所有权

  1. Rust 中每一个值都被一个变量所拥有，该变量被称为值的所有者
  2. 一个值同时只能被一个变量所拥有，或者说一个值只能拥有一个所有者
  3. 当所有者（变量）离开作用域范围时，这个值将被丢弃(drop)

rust永不自动创建数据的深拷贝。

rust中有个copy特性，一个旧的变量在被赋值给其他变量后仍然可用，赋值的过程就是copy的过程。

  • 所有整数类型，比如 u32
  • 布尔类型，bool，它的值是 true 和 false
  • 所有浮点数类型，比如 f64
  • 字符类型，char
  • 元组，当且仅当其包含的类型也都是 Copy 的时候。比如，(i32, i32) 是 Copy 的，但 (i32, String) 就不是
  • 不可变引用 &T ，例如转移所有权中的最后一个例子，但是注意：可变引用 &mut T 是不可以 Copy的

引用和借用

如果仅仅支持通过转移所有权的方式获取一个值，会让程序变的复杂。

引用和解引用

用&符号表示引用，用*解引用。引用不会发生所有权的转移。

let s = &s1;

如果要利用借用去改变值，需要把变量变成可变的mut。

  • 同一作用域，特定数据的可变引用只能存在一个，以下代码是错误的例子。原因在于第一个可变借用必须持续到println（最后一次使用的位置）。中间能有其他的s的可变引用的创建。
let mut s = String::from("hello");

let r1 = &mut s;
let r2 = &mut s;
println!("{}, {}", r1, r2);

我们可以使用{}来限制作用域，

let mut s = String::from("hello");

{
    let r1 = &mut s;

} // r1 在这里离开了作用域，所以我们完全可以创建一个新的引用

let r2 = &mut s;
  • 可变引用于不可变引用不能同时存在

​ 引用的作用域是持续到它最后一次使用的地方，变量是遇到}结束。NLL优化。

字符串和切片

1、切片

  • 切片允许引用集合中部分连续的元素序列，而不是引用整个集合。语法为【开始索引..终止索引】。其中终止索引比较特殊，他指向最后一个元素的后面一个元素的位置。索引值必须落在字符的边界位置
let s = String::from("hello world");

let hello = &s[0..5];
let world = &s[6..11];

  • 字符串字面量也是切片。&str类型。
  • 字符串的本质是一个u8类型的数组。rust不允许用索引的方式去访问字符串。

操作字符串

  • 追加（push）

  • 插入（insert）

  • 替换（replace）

  • 删除（delete）

  • 连接（+=）右边的参数必须是&slice类型。

  • format！

    fn main() {
    let s1 = "hello";
    let s2 = String::from("rust");
    let s = format!("{} {}!", s1, s2);
    println!("{}", s);
}

元组是由多种类型组合到一起形成的，长度固定，顺序固定。

可以用**.**访问元组。

unsafe语法

在rust语言中，unsafe是开发者和编译器之间的约定，用于处理编译器无法验证安全性的底层操作。需要程序员手动的去保证代码的安全性。

？运算符

示例： let policy = thread_schedule_policy()?;

? 是 Rust 中的错误传播运算符，用来简化对错误 (Result 或 Option) 的处理。

如果 thread_schedule_policy() 返回一个 Result<T, E> 类型：

  • 如果是 Ok(T)，? 会提取 Ok 中的值，并将其赋给变量 policy。
  • 如果是 Err(E)，? 会提前返回错误，不会继续执行当前函数。
