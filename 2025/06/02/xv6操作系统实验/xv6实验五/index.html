基础

页面错误异常

在risc-v中，有三种不同的页面错误：

  • 加载页面错误
  • 存储页面错误
  • 指令页面错误

当发生这些页面错误的时候，错误信息会保存在scause（页面错误类型）寄存器和stval（错误的虚拟地址）寄存器中，并且引起page fault的程序计数器的值也会记录到sepc中。

写时复制（cow）

写时拷贝（Copy On Write） 使得父子进程在 fork() 时几乎不需要做实际内存的复制，除非有写操作才真正发生复制。

如果子进程随即调用 exec()，那么父进程的地址空间更是用不着复制给它，直接换成新程序就行了，更进一步节省资源。

实验一：Eliminate allocation from sbrk() (easy)

第一个实验要求比较简单，删除sbrk(n)系统调用中的页面分配代码（位于***sysproc.c***中的函数sys_sbrk()）。sbrk(n)系统调用将进程的内存大小增加n个字节，然后返回新分配区域的开始部分（即旧的大小）。新的sbrk(n)应该只将进程的大小（myproc()->sz）增加n，然后返回旧的大小。它不应该分配内存——因此您应该删除对growproc()的调用。

在sys_sbrk函数中,添加

  struct proc* p  = myproc();
  
  if(n > 0){
    p->sz += n;  
  } else if(p->sz + n > 0) {
    p->sz = uvmdealloc(p->pagetable, p->sz, p->sz + n);
  }
  else
    return -1;

实验2： Lazy allocation (moderate)

要求：修改***trap.c***中的代码以响应来自用户空间的页面错误，方法是新分配一个物理页面并映射到发生错误的地址，然后返回到用户空间，让进程继续执行。您应该在生成“usertrap(): …”消息的printf调用之前添加代码。你可以修改任何其他xv6内核代码，以使echo hi正常工作。

分析：实验一只是实现了不分配内存只增加进程的大小的功能，但是在usertrap中会报错，所以需要对usertrap进行修改，让它能够处理这种异常。

因为可以通过r_scause函数可以获取异常类型，r_stval可以读取引发异常的虚拟地址。还要判断该虚拟地址是否合法：

在usertrap函数中添加：

实验3：Lazytests and Usertests

还需要实现uvmshoulddallocate函数，用来检测该虚拟地址是不是因为惰性分配还没分配和映射地址。uvmlazyallocate函数：给虚拟地址分配内存；

void lk_uvmlazyallocate(uint64 va) {
  struct proc* p = myproc();
  char* pa = kalloc();    //分配物理地址
  if (pa == 0) {
      printf("lazy alloc: out of memory\n");
      p->killed = 1;
  }
  else {
      memset(pa, 0, PGSIZE);
      if (mappages(p->pagetable, PGROUNDDOWN(va), PGSIZE, (uint64)pa, PTE_W | PTE_X | PTE_R | PTE_U) != 0) {      
          printf("lazy alloc: failed to map page\n");
          kfree(pa);
          p->killed = 1;
      }
  }
}

int lk_uvmshoulddallocate(uint64 va) {
  pte_t* pte;
  struct proc* p = myproc();

  return va < p->sz                   
      && PGROUNDDOWN(va) != r_sp()    
      && (((pte = walk(p->pagetable, va, 0)) == 0) || ((*pte & PTE_V) == 0));    
}

扩展

zero-fill-on-demand

在elf文件中，存在一个特殊的段bss（里面的内容全为0），为了节省elf文件文件的大小，所以只记录bss段的大小，不实际分配elf文件的空间。当执行exec时，将elf文件装载到虚拟地址空间，bss段可能分配了多个虚拟的page，但我们为了节省内存，可能会将这些vpage映射到同一个物理页，并设置权限为只读。当用户程序试图去修改时，就会触发page fault并创建新的物理页。
